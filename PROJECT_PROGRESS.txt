==============================================================
WAVEX - AI DECISION OPERATING SYSTEM FOR STARTUP FOUNDERS
Project Progress Log (Hackathon)
==============================================================

WHAT WAVEX IS
--------------------------------------------------------------
Not a chatbot. A decision operating system: founders ask business
questions, specialized AI experts analyze them using the company's
own data, a Judge issues a structured recommendation, and every
decision is remembered forever. The system gets smarter about the
company with every decision logged.

ARCHITECTURE - THE THREE BRAINS
--------------------------------------------------------------
Brain 1 - Business Context (SQLite via SQLAlchemy, Postgres-ready)
  - companies, documents, decisions tables
  - decisions table = the Decision Ledger (permanent audit trail)
  - Data survives restarts; swap to PostgreSQL = one env var

Brain 2 - Business Intelligence (RAG on ChromaDB)
  - Uploaded docs (PDF/TXT/MD) -> chunked (250 words, 40 overlap)
    -> embedded locally (all-MiniLM-L6-v2, no API key) -> indexed
  - Metadata filtering: company_id (privacy), domain (routing)
  - Global knowledge bases seeded at startup:
      * Legal KB: 9 chunks (India-focused: incorporation, ESOP,
        DPDP/GDPR, employment, contracts, fundraising, IP, compliance)
      * Finance KB: 4 chunks (runway/burn benchmarks, hiring costs,
        unit economics, fundraising norms)

Brain 3 - Decision Intelligence (Groq LLM agents)
  - Planner: selects ONLY relevant experts per question
    (keyword fallback if LLM unavailable)
  - Experts:
      * FinanceAgent  - KNOWLEDGE agent, grounded in finance KB
      * LegalAgent    - KNOWLEDGE agent, grounded in legal KB
      * HiringAgent   - ACTION agent, produces action_output
        (job descriptions, hiring plans)
      * GTMAgent      - ACTION agent, produces action_output
        (GTM plans, outreach sequences)
  - Judge: synthesizes into recommendation / why / trade-offs /
    next steps / confidence
  - Every agent degrades gracefully to fallback output if the
    LLM is unavailable (demo never crashes)

KEY FEATURES BUILT
--------------------------------------------------------------
1. Company profiles + document upload into private knowledge base
2. RAG-grounded expert analysis (agents cite the founder's own
   documents and the knowledge bases by name)
3. Specific agent selection (Planner routes legal questions to
   Legal only, hiring+budget to Hiring+Finance, etc.)
4. Action agents produce ready-to-use deliverables (action_output)
5. Decision Ledger: every decision persisted with full analysis
6. DECISION MEMORY with re-evaluation:
   - Approved/rejected decisions indexed into vector memory
   - New similar question -> response includes similar_past_decision
     (decision id, title, status, days_ago, similarity score)
   - Judge explicitly re-evaluates in light of the past decision
     and time elapsed
7. Database firewall:
   - API_KEY env -> all data routes require X-API-Key header (401
     otherwise); /health reports firewall status
   - CORS locked to whitelisted origins
   - ORM-parameterized queries (SQL injection blocked)
   - Multi-tenant isolation (company A can never retrieve
     company B's data - tested)

API ENDPOINTS
--------------------------------------------------------------
POST /company                          create company
POST /company/{id}/documents           upload doc into knowledge base
GET  /company/{id}/documents           list documents
POST /decision                         run full decision pipeline
GET  /decision/{id}                    fetch one decision
GET  /history                          decision ledger
POST /approve                          approve/reject (+ index into memory)
GET  /health                           health + firewall status

TECH STACK
--------------------------------------------------------------
FastAPI + Pydantic | SQLAlchemy + SQLite (PostgreSQL-ready)
ChromaDB (embedded vector DB) | all-MiniLM-L6-v2 local embeddings
Groq (llama-3.3-70b-versatile) for agent inference
pypdf for PDF extraction | React frontend (separate branch)

HOW TO RUN
--------------------------------------------------------------
cd backend
python -m pip install -r requirements.txt
copy .env.example .env   (add GROQ_API_KEY; optional API_KEY
                          to enable the firewall)
python -m uvicorn app.main:app --reload
Open http://127.0.0.1:8000/docs
NOTE: .env changes require a manual server restart.
First document upload downloads the embedding model once (~80MB).

WHY NOT JUST USE CLAUDE/CHATGPT ($20 ARGUMENT)
--------------------------------------------------------------
A chat LLM is a brilliant consultant with amnesia. WaveX is a
system of record: (1) Decision Ledger - auditable history of every
recommendation and approval; (2) organizational memory - answers
grounded in YOUR documents and YOUR past decisions, improving over
time; (3) structured output - recommendation/why/trade-offs/next
steps/confidence, every time; (4) multi-agent disagreement surfaces
trade-offs a single model blends away. The LLM is swappable;
the memory and workflow are the product.
